# Automatic Mode `beta`

Automatic mode allows you to use Million.js without any configuration. It will automatically determine which components are slow and optimize them.

## Installation

To enable automatic mode, you need to pass the `auto` option to the compiler.

<Tabs items={['Next.js', 'Astro', 'Gatsby', 'Vite', 'Create React App', 'Webpack', 'Rollup']} storageKey="selected-bundler-compiler">
  <Tab>

  ```js filename="next.config.mjs"
  import million from 'million/compiler';

  /** @type {import('next').NextConfig} */
  const nextConfig = {
    reactStrictMode: true,
  };

  export default million.next(nextConfig, { auto: true });
  ```

  </Tab>

  ```js filename="astro.config.mjs"
  import { defineConfig } from 'astro/config';
  ```

  </Tab>

  ```js filename="gatsby-node.js"

  exports.onCreateWebpackConfig = ({ actions }) => {
  ```

That's literally all you have to do.

## Ignoring components

If a certain component conficts with the Million.js runtime, it's possible to skip over them via the `// million-ignore` comment.

```js
// million-ignore
function App() {
  return ...
}
```

## React Server Components support

React Server Components require a bit of extra configuration to work with Million.js. You need to pass the `rsc` option into the `auto` option in order to enable it.

```js
{
  auto: { rsc: true }
}
```

## Muting help messages

If you don't want to see the help messages, you can pass the `mute` option to the compiler.

```js
{
  mute: true,
  auto: true
}
```
