# Automatic Mode

Million.js comes with Automatic Mode as its default setting. This transforms your React components into Million.js components, improving speed without any major code changes.


When [installing Million into your project](/docs/install), automatic mode is recommended for most users. This passes the `auto` option into the compiler.



### `auto`

The `auto` option is a boolean that enables automatic mode. By default it is unset, because we still support [manual mode](/docs/install#configure-it-yourself).


### Advanced customization

Automatic mode provides customization options:

- `threshold` (default: `0.1`): What is used to determine whether a component should be converted to Million.js. The greater the threshold, fewer components will be optimized, and vice versa.
- `skip` (default: `[]`): An array of identifiers to indicate if a component should be skipped. You can use hook names, variable names, etc.

```js
const options = {
  auto: {
    threshold: 0.25, // default: 0.1,
    skip: ['useBadHook', /badVariable/g] // default []
  }
}
```


When using <a className="font-normal text-base" href='https://vitejs.dev/guide/'>Vite</a>, the `auto` object lets you configure options beyond the default ones set during installation:

```js
export default defineConfig({
  plugins: [million.vite({auto: {
    threshold: 0.25, // default: 0.1,
    skip: ['useBadHook', /badVariable/g] // default []
}}), react()],
});
```


### Ignoring components

If you encounter an error with a component during the Million.js runtime, it can be skipped using the `// million-ignore` comment.

```js
// million-ignore
function App() {
  return ...
}
```



### Muting help messages

To avoid seeing help messages messages, you can pass the `{ mute: true }{:js}` option to the compiler.
